## Story
**As a** novo usuário,
**I want** responder um quiz de saúde e objetivos passo a passo,
**so that** o sistema entenda meu perfil e possa criar treinos adequados e seguros para mim.

## Acceptance Criteria
1. Fluxo de 10 perguntas apresentadas uma por vez com barra de progresso visível
2. Perguntas na ordem: (1) Peso, (2) Idade, (3) Altura, (4) Tempo sem praticar atividade, (5) Doença cardíaca, (6) Lesão crônica, (7) Medicamento controlado, (8) Objetivo principal, (9) Dias por semana, (10) Preferência de treino
3. Validação de cada campo antes de permitir avançar (campo obrigatório, valores numéricos válidos)
4. Botão "Voltar" funcional para corrigir respostas anteriores
5. Dados salvos no banco (tabela `user_profiles`) ao concluir o quiz
6. Usuário que já completou o onboarding não vê o quiz novamente
7. Design mobile-first com teclado numérico automático em campos de número
8. Tela de transição/loading entre concluir e ir para a próxima etapa

## Dev Notes

### Zustand Store para Onboarding
```typescript
// src/stores/onboarding.store.ts
import { create } from 'zustand'

interface QuizAnswers {
  peso?: number
  idade?: number
  altura?: number
  tempo_sem_treinar?: string
  doenca_cardiaca?: boolean
  lesao_cronica?: boolean
  lesao_descricao?: string
  medicamento_controlado?: boolean
  objetivo?: string
  dias_por_semana?: number
  preferencia_treino?: string
}

interface OnboardingStore {
  step: number
  answers: QuizAnswers
  setStep: (step: number) => void
  setAnswer: (key: keyof QuizAnswers, value: unknown) => void
  reset: () => void
}

export const useOnboardingStore = create<OnboardingStore>((set) => ({
  step: 0,
  answers: {},
  setStep: (step) => set({ step }),
  setAnswer: (key, value) => set((state) => ({ answers: { ...state.answers, [key]: value } })),
  reset: () => set({ step: 0, answers: {} }),
}))
```
